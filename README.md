## 📂 Структура файлів

Кожен розробник працює з двома окремими файлами, назва яких відповідає назві
вашої секції:

- `назва-секції.html` — HTML-розмітка вашої частини.
- `назва-секції.css` — CSS-стилі вашої частини.

---

## 🏗 Структура HTML-розмітки

Розмітка кожної секції повинна мати таку ієрархію:

1.  **Кореневий тег** — тег `<section>`.
2.  **Контейнер** — першим вкладеним елементом у секції завжди йде
    `<div class="container">`.
3.  **Додаткові класи** — цьому діву (`.container`) можна задати додатковий клас
    для налаштування верхніх та нижніх падінгів (наприклад,
    `container--padding-lg`).

**Приклад:**

```html
<section>
  <div class="container section-hero-padding">
    <!-- Далі йде внутрішня структура секції -->
  </div>
</section>
```

---

## 🎨 Правила іменування CSS-класів

Ми використовуємо систему префіксів, щоб уникнути конфліктів у стилях.

1.  **Специфічні класи:** Усі класи всередині вашої секції повинні починатися з
    назви цієї секції.
    - ✅ _Правильно:_ `<div class="hero-box"></div>`,
      `<p class="footer-text"></p>`
2.  **Загальні класи (Global):** Такі класи, як `.container`, залишаються без
    змін.
    - ⚠️ **ВАЖЛИВО!** До назв загальних класів назва секції **не додається**.
    - 🚫 **ЗАБОРОНЕНО** переписувати або змінювати код загальних класів у своїх
      файлах!

## 📱 Методологія Mobile First

Розробка ведеться суворо в такому порядку:

1.  **Мобільна версія** (базові стилі).
2.  **Планшетна версія** (Media queries).
3.  **Десктопна версія** (Media queries).

> _Спочатку створюємо стилі для найменших екранів, а потім додаємо зміни для
> більших._
